ngeo

JavaScript toolkit combining OpenLayers 3 and AngularJS, aimed at creating rich web-mapping applications

  • For developers who use OpenLayers and Angular for creating web-mapping apps
  • OpenLayers 3: open-source JS lib for adding maps to web page
  • Angular: open-source JS framework created by Google
  • Goal is to ease development
  • ngeo is open-source

Features

Targeted features include:

  • Layer tree
  • Measure tools
  • Elevation profile
  • …

  • ngeo provides both low-level and high-level components
  • Examples shown later

Design principle

Provide atomic components that can be combined in many different ways

  • ngeo should not get in your way
  • ngeo should constrain you
  • for example ngeo should not limit you in the way you can use OpenLayers 3
  • other ol3/ng libs exist, what does make ngeo unique?
  • flexibility, modularity
  • the primary goal is not to provide out-of-the-box widgets
  • you can style your components the way you want
  • ngeo is not tied to a specific UI toolkit (although we use it mainly with Bootstrap)

The ngeo approach

→We're not looking at creating a declarative layer/API above OpenLayers 3!

  • We want developer keeps maximum control of his code.
  • Don't want to embed features in too heavy/complex markup.
  • Easier to understand and maintain.

Closure Compiler

We use Closure Compiler, and recommend using it in ngeo applications 


/**
 * @param {string|function(!angular.JQLite=, !angular.Attributes=)}
 *     ngeoScaleselectorTemplateUrl Template URL for the directive.
 * @return {angular.Directive} Directive Definition Object.
 * @ngInject
 */
ngeo.scaleselectorDirective =
    function(ngeoScaleselectorTemplateUrl) {
  return {
    restrict: 'A',
    scope: true,
    controller: 'NgeoScaleselectorController',
    templateUrl: ngeoScaleselectorTemplateUrl
  };
};

  • Closure Compiler helps us structure, document and maintain our code
  • Type checker and code compressor/minifier
  • Part of our guidelines
  • Make it possible to create toolkit with lots of features
